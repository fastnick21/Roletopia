BUILD WITH CLARITY

A practical overview of the current repository, supported targets, development workflow, and the boundaries between portable framework code and future Among Us runtime integration.

01 — OVERVIEW

Current project status

Roletopia has a portable framework, role registry, game-state model, network abstraction, plugin-manifest discovery, a Windows installer project, smoke tests, and a multi-page website. It does not yet contain the BepInEx/IL2CPP game entry point required to run inside Among Us.

What “portable” means

The reusable libraries target .NET Standard 2.0 so they can build and be tested on Windows and Linux. This does not mean the game mod itself can be installed on every platform.

02 — ARCHITECTURE

Repository components

Core Engine

Tracks players, alive and connected state, tasks, meetings, votes, ties, skips, and ejection results.

Role System

Defines the role catalog, behavior interface, registry, and validation boundary for future abilities.

Networking

Provides a thread-safe outbound packet queue and an abstract transport to be implemented by the game layer.

Plugin Loader

Discovers and validates JSON manifests while rejecting duplicate or malformed plugin identifiers.

Installer

A Windows console installer that locates the Steam game folder and copies packaged mod files.

Game Integration

Not implemented yet. This module must connect BepInEx, IL2CPP, Harmony patches, UI, and Among Us RPC calls.

03 — BUILD LOCALLY

Development commands

Install the .NET 8 SDK, clone the repository, and run the solution build from the repository root.

git clone https://github.com/fastnick21/Roletopia.git
cd Roletopia
dotnet restore Roletopia.sln
dotnet build Roletopia.sln --configuration Release

The GitHub Actions workflow also builds portable projects and runs smoke tests on Windows and Ubuntu. The Windows installer is built separately.

04 — PLUGINS

Plugin manifest format

The current loader discovers files ending in .plugin.json. A manifest describes identity and declared roles; executable plugin loading and a stable SDK are future work.

{
  "pluginId": "creator.example-role-pack",
  "name": "Example Role Pack",
  "version": "0.1.0",
  "roles": ["ExampleRole"]
}

Validation rules

Use a unique non-empty plugin ID, a readable name, a parseable version, and non-duplicate role names. A valid manifest does not guarantee that a plugin DLL is safe or compatible.

05 — COMPATIBILITY

What is and is not supported

  • Portable libraries: Windows and Linux build targets.
  • First mod host target: Windows 10/11 with Among Us on Steam.
  • Linux/Steam Deck host: future investigation after the Windows integration works.
  • Vanilla mobile and console clients: a design goal, not yet verified.
  • Installing the mod on consoles or mobile: not supported.

Compatibility must be tested against specific Among Us versions. Game updates can change IL2CPP classes, RPC behavior, UI methods, and loader compatibility.

06 — CONTRIBUTING

Help without hiding unfinished work

Open an issue with reproducible steps, operating system, game version, expected behavior, actual behavior, and relevant logs. Keep pull requests focused and ensure portable tests pass.
